# ADR 0008 — PBKDF2-HMAC-SHA256 para hash de senha

- **Data:** 2026-05-16
- **Status:** Aceita

## Contexto

Versão anterior usava `SHA-256(senha + salt)` puro — hash criptográfico genérico,
projetado pra ser **rápido**. GPUs e ASICs quebram senhas comuns em segundos.
Pra senha humana, queremos o oposto: hash **deliberadamente lento** (key stretching).

## Alternativas avaliadas

| Algoritmo | Status |
|---|---|
| **SHA-256 puro** (atual) | ❌ Inviável pra produção. ~1µs por hash → GPU testa bilhões/s. |
| **bcrypt** | ⚠️ Bom histórico (1999), mas truncado em 72 bytes e não tem flag de memory-hardness. |
| **PBKDF2-HMAC-SHA256** | ✅ **Escolhido**. Nativo no .NET (`Rfc2898DeriveBytes.Pbkdf2`), RFC 2898, OWASP-aprovado, sem dependência externa. |
| **Argon2id** | ⭐ Estado da arte (PHC winner 2015). Resistente a GPU/ASIC. Mas exige NuGet `Konscious.Security.Cryptography`. **Sprint 5** se for promovido a prod. |

## Decisão

PBKDF2-HMAC-SHA256, **100.000 iterações**, hash 32 bytes, salt 16 bytes.

### Parâmetros

- **100k iterações**: ~10ms por hash em CPU moderna. OWASP recomenda mínimo 600k
  pra ambientes high-stakes, mas 100k é o mínimo prático que ainda inviabiliza
  bruteforce sem deixar login lento demais.
- **HMAC-SHA256**: variante mais forte do PBKDF2 (vs SHA-1, vulnerável).
- **CryptographicOperations.FixedTimeEquals** na verificação — previne timing attacks.

### Formato self-describing

Hash gravado como `pbkdf2$<iter>$<saltB64>$<hashB64>`. O verify decodifica os
parâmetros do hash gravado (não usa os atuais). Permite **mudar iterações sem
migração** — pode-se subir pra 200k pra contas novas sem invalidar antigas.

### Compatibilidade legacy

Hashes pré-PBKDF2 (formato hex sem prefixo) são tratados como SHA-256 legacy no
`VerifyPassword`. Migra implicitamente: futuro `LoginHandler` pode reescrever o
hash após login bem-sucedido (Sprint 5).

## Consequências

- ✅ Cracking de senha bem mais caro: 10ms × 10⁹ tentativas = 116 dias por senha
  comum (vs segundos com SHA-256 puro).
- ✅ Sem dependência externa (`Rfc2898DeriveBytes` é System.Security.Cryptography).
- ✅ Backward compat: contas antigas (Alice, Bob) ainda logam com SHA-256 legacy.
- ⚠️ Login ~10ms mais lento — imperceptível pro usuário, mas multiplicar por 100
  req/s de login = 1 core sustentado. Pra alta escala, considerar cache de
  verificação por session ou subir pra Argon2id com memory-hardness.

## Verificação

```bash
curl -X POST http://localhost:5000/api/contacorrente/criar \
  -H "Content-Type: application/json" \
  -d '{"nome":"X","cpf":"<cpf-valido>","senha":"abc123","saldoInicial":0}'

docker exec bankmore-postgres psql -U bankmore -d bankmore_db \
  -c "SELECT LEFT(senha, 25) FROM contacorrente WHERE cpf='<cpf>';"
# Esperado: pbkdf2$100000$<base64>
```
